## Start from constraints

System design improves when we start with constraints: team size, delivery speed, and product scope.

## Keep boundaries clear

For frontend projects, good boundaries between UI primitives, feature modules, and data adapters reduce chaos.

## A simple review checklist

1. Is this reusable or feature-specific?
2. Is the API for this module clear?
3. Can this be replaced later without breaking consumers?
